1. 🧵 Product Variants Matrix (Size × Color Grid)
Priority: CRITICAL

What's missing: The ProductSKU model has color and size as free-text fields. There is no proper variant matrix — the grid that shows all combinations of Size (S/M/L/XL) × Color (Red/Blue/Black) as a visual table.

Real-life impact:

A buyer cannot select "Blue L" vs "Blue M" from a single product
Reporting can't show "XL selling faster than S" without this structure
No way to prevent duplicate variant combinations
What's needed:

ProductTemplate → ProductAttribute (Size, Color, Fabric) → AttributeValues
ProductVariant with variant_combination as unique constraint
Visual Variant Matrix UI (the grid/spreadsheet view for stock entry)
Variant-level pricing (XL may cost more than S)
2. 📄 GST Compliance & Tax Engine
Priority: CRITICAL (India-specific)

What's missing: The Order model has tax_type and tax_amount fields, but there is no actual GST calculation engine — no HSN codes, no CGST/SGST/IGST split, no B2B vs B2C differentiation.

Real-life impact:

Every invoice is legally invalid without correct GST
Cannot file GSTR-1 / GSTR-3B without structured tax data
Garments have different GST slabs: 5% (< ₹1000), 12% (> ₹1000)
What's needed:

HSN Code field on ProductSKU
GST rate slab engine (5% / 12% / 18% based on value)
CGST + SGST vs IGST split based on customer state
B2B invoice with customer GSTIN
GSTR-1 export (JSON format for filing)
E-Invoice (IRN generation for B2B > ₹5 Cr turnover)
3. 🏭 Manufacturing / Job Work Module
Priority: HIGH (Critical if you do in-house stitching)

What's missing: No manufacturing model at all. Phase 9 in your implementation list covers this but nothing has been built.

Real-life impact:

Cannot track fabric consumption → finished garment conversion
Cannot manage subcontracting (cut → stitch → finish)
No Bill of Materials (BOM) to know raw material cost per piece
Cannot track work-in-progress (WIP) inventory
What's needed:

BillOfMaterials model (Fabric: 1.5m + Thread: 50m + Buttons: 6 → 1 Shirt)
WorkOrder / ProductionOrder model
Raw material reservation on production start
Subcontracting flow (job work to external tailor)
Fabric defect tracking (QC before cutting)
WIP stock locations (Cutting Stage, Stitching Stage, Finishing)
4. 📦 Bulk Import / Excel Upload
Priority: HIGH

What's missing: No bulk product upload endpoint or UI. For a garment business with 500-10,000 SKUs (per season), manual entry is not feasible.

Real-life impact:

Adding a new season's catalog (200 styles × 5 colors × 5 sizes = 5,000 variants) manually is impossible
No way to update prices in bulk before season change
Cannot import opening stock from Excel
What's needed:

Excel/CSV upload endpoint for Products, Variants, Stock, Prices
Validation with error report (show which rows failed and why)
Template download with correct column headers
Duplicate SKU detection on upload
Async processing via Celery for large files
5. 📊 Business Intelligence & Reporting
Priority: HIGH

What's missing: analytics.py is only 2.4KB. The existing "Financial Intelligence" page calls one endpoint. No garment-specific reports exist.

Real-life impact:

Cannot answer: "Which size is selling most this season?"
Cannot see dead stock (items with 0 sales in 90 days)
Cannot compare revenue by brand/category/supplier
No P&L per SKU (needed for pricing decisions)
What's needed:

Size/Color Sales Matrix — which combinations sell and which don't
Dead Stock Report — items unsold for 30/60/90 days with ageing value
Fast Movers vs Slow Movers — Pareto / ABC analysis
Season vs Season comparison (Summer 2024 vs Summer 2025)
Gross Margin per SKU (selling price − cost price × qty sold)
Supplier-wise purchase analysis (who delivered on time, who didn't)
Customer purchase frequency (loyalty & repeat rate)
Excel export for all reports
6. 🔄 Season & Collection Management
Priority: HIGH (Garment-specific)

What's missing: No Season or Collection model exists. Your implementation plan (Phase 12.3) lists this but it hasn't been built.

Real-life impact:

Cannot tag products to Summer/Winter/Festive collections
Cannot archive last season's catalog cleanly
Cannot run "End of Season Sale" with automatic markdowns on aged stock
Reports cannot filter by season
What's needed:

Season model (Summer 2025, Winter 2024, Festive 2025)
Collection tags per product/variant
Season-based price override (MRP vs season sale price)
End-of-season auto-discount workflow
Season close → move unsold stock to clearance location

Roles: Owner, Manager, Cashier, Warehouse Staff, Accountant

9. 💳 Loyalty & Credit Management
Priority: MEDIUM

What's missing: Coupons exist but no loyalty points, no store credit, no credit limit for wholesale buyers.

Real-life impact:

What's needed:

Loyalty points earn/redeem (1 point per ₹10 spent)
Customer credit limit (for wholesale/B2B accounts)
Aging report for outstanding credit (who owes how much)
Store credit for returns (instead of cash refund)
10. 📝 Physical Stock Count (Cycle Count)
Priority: MEDIUM

What's missing: The InventoryCount and CountLine models exist in the database but there is zero UI or API endpoint for conducting a physical stock count.

Real-life impact:

Cannot do monthly/quarterly physical verification
System stock and actual stock will diverge over time
Shrinkage (theft/loss) is invisible
What's needed:

Create a stock count session (select warehouse/location)
Print a count sheet (barcode + product + system qty)
Enter counted qty via barcode scan or manual entry
System generates variance report (system qty vs counted qty)
One-click adjustment to reconcile differences
11. 🚚 Delivery Challan & E-Way Bill
Priority: MEDIUM (Indian compliance)

What's missing: Invoices exist but no Delivery Challan (DC) or E-Way Bill generation.

Real-life impact:

Moving goods worth > ₹50,000 without E-Way Bill is illegal under GST
Deliveries to wholesale buyers need a Delivery Challan
Stock transfers between warehouses need a DC
What's needed:

Delivery Challan template (against sale order or stock transfer)
E-Way Bill JSON generation (for NIC portal API)
Vehicle number + transporter details on shipment
DC print/PDF download
